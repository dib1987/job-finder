# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Architecture: WAT (Workflows, Agents, Tools)

```
Layer 1: Workflows  — workflows/job_application.md   (WHAT to do)
Layer 2: Agent      — agent.py                       (WHEN and sequencing)
Layer 3: Tools      — tools/                         (HOW — deterministic executors)
```

The agent orchestrates; tools execute. Tools never call each other. Tools never call the agent.

`agentic_base/` is a **sibling directory** at `c:\Agentic Workflow\agentic_base` — not inside this project. `agent.py` adds it to `sys.path` at startup. Other agents (HospitalService, SocialMedia) share this same base.

## Running the Agent

```bash
# Full pipeline
python agent.py

# Single phase (resumes from .tmp/ state)
python agent.py --phase parse
python agent.py --phase scrape
python agent.py --phase score
python agent.py --phase approve
python agent.py --phase apply
python agent.py --phase notify

# Dry run — full pipeline, no actual submissions
python agent.py --dry-run

# LinkedIn session setup (one-time, opens browser)
python agent.py --setup-linkedin

# View history / stats
python agent.py --history
python agent.py --stats
```

**Windows note:** Run with `PYTHONUTF8=1` prefix if emoji print statements cause encoding errors (cp1252 limitation).

## Key Environment Variables

All in `.env` (copy from `.env.example`):

| Variable | Default | Purpose |
|---|---|---|
| `ANTHROPIC_API_KEY` | — | Required. Primary LLM |
| `LLM_MODEL` | `claude-sonnet-4-6` | Swap to `gpt-4o`, `gemini-1.5-pro`, etc. |
| `JOB_SOURCE` | `linkedin_scraper` | Switch to `rapidapi` — zero code changes |
| `SCORING_THRESHOLD` | `70` | Override without editing preferences.json |
| `MAX_JOBS_PER_RUN` | `30` | Keep ≤30 to reduce LinkedIn ban risk |
| `LINKEDIN_SESSION_PATH` | `config/linkedin_session.json` | Auto-generated by `--setup-linkedin` |

## Inter-Phase State

Phases pass data to each other via `.tmp/` files (gitignored). Each file is named `{key}_{YYYYMMDD}.json`:

```
.tmp/parsed_resume_{date}.json
.tmp/scraped_jobs_{date}.json
.tmp/scored_jobs_{date}.json
.tmp/jobs_above_threshold_{date}.json
.tmp/approved_jobs_{date}.json
.tmp/apply_results_{date}.json
.tmp/run_summary_{date}.json
```

Running `--phase score` expects `parsed_resume` and `scraped_jobs` to already exist in `.tmp/`. If they're missing, the phase exits with an actionable error message.

## Key Design Decisions

### 1. Job Source Abstraction
`tools/job_source/base.py` defines the `JobSource` ABC and `JobListing` dataclass — this is the primary extension seam. `agent.py` only ever calls `source.fetch_jobs()`. To swap sources: set `JOB_SOURCE=rapidapi` in `.env`. Zero agent code changes.

### 2. LLM Abstraction
`agentic_base/utils/llm_client.py` provides `get_llm_client()` which reads `LLM_MODEL` from env. Same interface regardless of provider — Claude, OpenAI, or Gemini.

### 3. Human-in-the-loop (non-negotiable)
`tools/approval_tool.py` gates every application. `auto_apply_above` in `preferences.json` allows bypassing for scores ≥90 — opt-in, off by default.

### 4. Resume Years Override
The PDF parser (`tools/resume_parser_tool.py`) can miscount years of experience from PDF dates. Set `personal.total_years_experience` in `preferences.json` to override the parsed value — `agent.py` applies this override in `phase_parse()`.

### 5. Duplicate Prevention
`tools/tracker_tool.py` checks SQLite (`data/applications.db`) before each apply. Daily runs will never re-apply to the same job.

### 6. No Resume Tailoring (POC)
`config/resume.pdf` is sent as-is. Resume tailoring is a deferred Phase 2 feature — `apply_tool.py` already accepts a custom resume path for when it's added.

## Extending This Agent

### Add a new job source (e.g., Indeed)
1. Create `tools/job_source/indeed_scraper.py`, implement `JobSource` ABC
2. Add to factory in `tools/job_source/base.py`:
   ```python
   if source_type == "indeed":
       from tools.job_source.indeed_scraper import IndeedScraper
       return IndeedScraper()
   ```
3. Set `JOB_SOURCE=indeed` in `.env`

### Add resume tailoring (Phase 2)
1. Create `tools/tailoring_tool.py`
2. Add `phase_tailor()` to `agent.py` and insert `"tailor"` into `PHASES` between `"score"` and `"approve"`
3. Pass tailored resume path to `apply_tool.py` instead of original

### Convert approval gate to web UI
1. Replace `_prompt_decision()` in `approval_tool.py` with a WebSocket callback
2. Agent blocks waiting for callback — same interface, no changes to `agent.py`

## Troubleshooting

| Problem | Fix |
|---|---|
| `Session expired` in scraper | `python agent.py --setup-linkedin` |
| `Resume not found` | Copy resume to `config/resume.pdf` |
| `ANTHROPIC_API_KEY not set` | Add key to `.env` |
| Scoring returns 0 for all jobs | Check LLM response in `logs/` — likely prompt format issue |
| Easy Apply button not found | Job doesn't support Easy Apply; marked `manual_apply_needed` in tracker |
| LinkedIn blocking (CAPTCHA) | Reduce `MAX_JOBS_PER_RUN`, increase delays in `linkedin_scraper.py` |
| Emoji UnicodeEncodeError on Windows | Prefix command with `PYTHONUTF8=1` |
