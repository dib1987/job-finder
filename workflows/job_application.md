# Job Application Workflow

## Purpose
Automate the end-to-end job application process: find relevant jobs on LinkedIn,
score them against the candidate's resume and preferences, get human approval,
and submit applications via LinkedIn Easy Apply.

## Trigger
- **Daily automated run**: Windows Task Scheduler fires `python agent.py` at 08:00
- **Manual run**: `python agent.py` or `python agent.py --phase <name>`

## Phases

### Phase 1: Parse Resume
**Goal**: Convert the candidate's PDF resume into structured JSON for scoring.

**Input**: `config/resume.pdf`
**Output**: `.tmp/parsed_resume_YYYYMMDD.json`

**What happens**:
- PyMuPDF extracts text preserving reading order
- Regex patterns extract: name, email, phone, location, summary, skills, experience, education
- Total years of experience computed from experience dates
- Raw text preserved alongside structured data (used by LLM scorer)

**Skip if**: `.tmp/parsed_resume_YYYYMMDD.json` already exists → run from phase 2.

---

### Phase 2: Scrape Jobs
**Goal**: Collect job listings from LinkedIn matching the candidate's target titles.

**Input**: `config/preferences.json` (target_titles, location, max_jobs_per_run)
**Output**: `.tmp/scraped_jobs_YYYYMMDD.json`

**What happens**:
- Playwright loads saved LinkedIn session (`config/linkedin_session.json`)
- Builds search URL from target titles + location preference
- Scrolls results page collecting up to `max_jobs_per_run` job IDs
- Visits each job detail page to extract full description, seniority, Easy Apply flag
- Random 2-4s delays between page loads (anti-detection)

**Error handling**:
- Session expired → logs error, stops phase, prompts: `python agent.py --setup-linkedin`
- Individual job fetch fails → skips that job, logs warning, continues

---

### Phase 3: Score Jobs
**Goal**: Rate each job 0-100 for fit using the candidate's resume and preferences.

**Input**: `.tmp/parsed_resume_YYYYMMDD.json` + `.tmp/scraped_jobs_YYYYMMDD.json`
**Output**: `.tmp/scored_jobs_YYYYMMDD.json` + `.tmp/jobs_above_threshold_YYYYMMDD.json`

**What happens**:
- Each job description + resume sent to LLM (Claude) with scoring rubric
- Rubric produces sub-scores: skills(30) + experience(25) + location(20) + compensation(15) + company(10) = 100
- JSON-only LLM response parsed into ScoringResult
- Jobs filtered by `scoring_threshold` (default: 70)
- Results sorted highest-score first

**LLM cost estimate**: ~$0.01–0.02 per job scored with Claude Sonnet. 30 jobs ≈ $0.30–0.60/day.

---

### Phase 4: Approve
**Goal**: Human reviews top matches and decides apply/skip before anything is submitted.

**Input**: `.tmp/jobs_above_threshold_YYYYMMDD.json`
**Output**: `.tmp/approved_jobs_YYYYMMDD.json`

**What happens**:
- Each job displayed with: score breakdown, matched skills, missing skills, rationale, URL
- User presses [A]pply, [S]kip, or [Q]uit
- Decisions saved immediately to `.tmp/approval_state_YYYYMMDD.json`
- If user quits mid-session: resume with `python agent.py --phase approve` (skips decided jobs)

**Manual apply jobs**: Jobs without Easy Apply are presented but flagged — agent prepares the
record but cannot submit. User applies on the company site themselves.

---

### Phase 5: Apply
**Goal**: Submit LinkedIn Easy Apply for all approved jobs.

**Input**: `.tmp/approved_jobs_YYYYMMDD.json` + `config/resume.pdf`
**Output**: `data/applications.db` (tracker updated)

**What happens**:
- For each approved job: checks tracker for duplicates → skips if already applied
- Opens job page in headless Playwright browser
- Clicks Easy Apply → fills personal info from `preferences.json` → uploads `resume.pdf`
- Steps through multi-page form → clicks Submit on final step
- Records result (applied/failed) in SQLite tracker

**DRY_RUN=true**: Skips actual submission, records as "dry_run" in tracker. Always test this way first.

---

### Phase 6: Notify
**Goal**: Email the candidate a summary of the run.

**Input**: All `.tmp/` state files from current run
**Output**: Email sent via Gmail SMTP + `output/run_reports/report_YYYYMMDD_HHMM.json`

**Email contains**:
- Jobs scanned / matches found / applied / pending your review
- Top 5 matches with scores and LinkedIn URLs
- Warning if pending approvals are waiting

---

## Configuration Reference

| File | Purpose |
|---|---|
| `config/preferences.json` | Job search preferences, personal info, scoring thresholds |
| `config/resume.pdf` | Original resume — submitted as-is to all approved jobs |
| `config/linkedin_session.json` | Saved browser auth (generated by --setup-linkedin) |
| `.env` | API keys, credentials, behavior flags |

## Key Commands

```bash
# First-time setup
python agent.py --setup-linkedin        # One-time LinkedIn login

# Full pipeline
python agent.py                         # Run all phases
python agent.py --dry-run               # Full run, no actual submissions

# Individual phases (for debugging)
python agent.py --phase parse
python agent.py --phase scrape
python agent.py --phase score
python agent.py --phase approve
python agent.py --phase apply --dry-run

# History & stats
python agent.py --history               # Application history table
python agent.py --stats                 # Summary statistics

# Scheduler
python scheduler.py --install --time 08:00   # Daily automation
python scheduler.py --status                  # Check if scheduled
```
